# Podcast Player on Briefing Pages — Design

**Date:** 2026-05-16
**Repo:** `~/Documents/profolio` (deployed to `https://yillonmask.github.io/briefings/`)
**Status:** Approved (ready for implementation planning)

## Goal

Add a per-date custom audio player UI to each briefing HTML page on the portfolio site, so visitors can listen to the AI-generated Mandarin audio digest matching that day's written briefing. The player must include play/pause, skip ±10 seconds, a click-to-seek progress bar, and current/total time display. It must match the existing site aesthetic (Fraunces serif, Geist mono, terracotta `#b8412c`, light/dark theme) and gracefully omit itself on briefing pages that have no MP3.

## Scope

**In scope**
- Detecting `briefings/audio/<DATE>-podcast.mp3` per briefing during the static build and conditionally rendering a player block in the HTML output.
- Player UI: play/pause button, skip −10s, skip +10s, click-to-seek progress bar with buffered indicator, M:SS / M:SS time display, AI-generation disclosure caption.
- Keyboard shortcuts: `Space` toggles play, `←`/`→` skip ±10s.
- Light/dark theme support via existing CSS variables.
- Mobile responsive layout (<720px breakpoint).
- `publish.sh` extension to copy the MP3 from the source `podcast/` directory into `briefings/audio/` before rebuilding.

**Out of scope (YAGNI)**
- Variable playback speed.
- Download MP3 link.
- Sticky player that persists across scroll or navigation.
- Episode list / player embed on `briefings/index.html`.
- Offline support / service worker / caching.
- Waveform visualization.
- Sharing / timestamp deep-linking.

## Architecture

Single-source-of-truth approach: the player markup, CSS, and JS all live in `build_briefings.py` and are injected at HTML render time. The markdown source files stay clean (no `<audio>` tags or directives). Player presence is determined entirely by file presence at build time.

```
~/Library/CloudStorage/.../podcast/
  └── 2026-05-14-podcast.mp3         (source MP3, generated by briefing_to_podcast.py)

~/Documents/profolio/                (portfolio repo, pushed to GitHub Pages)
  ├── publish.sh                      → extended to also copy MP3
  ├── build_briefings.py              → extended to detect MP3 + inject player
  └── briefings/
      ├── audio/                      → new dir, holds MP3s for GH Pages to serve
      │   └── 2026-05-14-podcast.mp3
      ├── 2026-05-14-briefing.md      (unchanged source)
      └── 2026-05-14-briefing.html    (player auto-injected because MP3 exists)
```

### Data flow for `./publish.sh 2026-05-14`

1. Copy `<OneDrive>/2026-05-14-briefing.md` → `briefings/` (existing behavior).
2. **New:** check for `<OneDrive>/podcast/2026-05-14-podcast.mp3`. If present, copy to `briefings/audio/`. If absent, skip silently.
3. Run `build_briefings.py`. For each `<DATE>-briefing.md`, the script checks `briefings/audio/<DATE>-podcast.mp3`; if present, the player block is injected between the brief-head header and the prose main.
4. `git add briefings/` (already recursive, picks up new MP3 and HTML). Commit. Push.

A date with a markdown briefing but no MP3 renders normally — no player, no errors. A date can get an audio backfill later by simply dropping the MP3 into `briefings/audio/` and rerunning the build.

## Component design

### Player HTML

Inserted between `<header class="brief-head">…</header>` and `<main class="prose">…`:

```html
<section class="brief-audio" aria-label="Audio digest">
  <div class="container">
    <div class="audio-card">
      <div class="audio-controls">
        <button class="audio-btn audio-back" aria-label="Back 10 seconds">
          <span class="audio-icon">⏪</span><span class="audio-num">10</span>
        </button>
        <button class="audio-btn audio-play" aria-label="Play">
          <span class="audio-icon-play">▶</span>
        </button>
        <button class="audio-btn audio-fwd" aria-label="Forward 10 seconds">
          <span class="audio-num">10</span><span class="audio-icon">⏩</span>
        </button>
        <div class="audio-track" role="slider" aria-label="Seek">
          <div class="audio-buffered"></div>
          <div class="audio-progress"></div>
        </div>
        <div class="audio-time">
          <span class="audio-cur">0:00</span> / <span class="audio-dur">--:--</span>
        </div>
      </div>
      <p class="audio-caption">AI-generated audio digest · Mandarin · Voiced by Fish Audio TTS</p>
      <audio class="audio-el" preload="metadata" src="audio/<DATE>-podcast.mp3"></audio>
    </div>
  </div>
</section>
```

`<DATE>` is interpolated by the build script.

### CSS additions to `PAGE_CSS`

All colors come from the existing CSS variables (`--bg`, `--card`, `--rule`, `--ink`, `--ink-soft`, `--ink-muted`, `--accent`), so dark theme is automatic.

- `.brief-audio` — wrapper section with top + bottom padding so the card breathes between the header rule and the prose. Padding `24px 0 8px`.
- `.audio-card` — `background: var(--card)`, `border: 1px solid var(--rule)`, `border-radius: 8px`, `padding: 20px 24px`. Same `760px` container width as the prose (already provided by `.container`).
- `.audio-controls` — `display: flex; align-items: center; gap: 14px;`
- `.audio-btn` — circular, 40px diameter, 1px `var(--rule)` border, background transparent, `color: var(--ink-soft)`, `cursor: pointer`, `transition: all 0.15s`. Hover/focus → `border-color: var(--accent); color: var(--accent);`. The play button (`.audio-play`) gets 48px diameter to be the visual anchor.
- `.audio-num` — Geist Mono, 11px, sits next to the chevron in the ±10s buttons.
- `.audio-track` — `flex: 1; height: 4px; background: var(--rule); border-radius: 2px; position: relative; cursor: pointer;`
- `.audio-buffered` — absolutely positioned, height inherited, `background: color-mix(in srgb, var(--accent) 25%, transparent);` width set via JS.
- `.audio-progress` — absolutely positioned, height inherited, `background: var(--accent)`, width set via JS.
- `.audio-time` — Geist Mono, 12px, `var(--ink-muted)`, `white-space: nowrap`.
- `.audio-caption` — Geist 12px italic, `var(--ink-muted)`, `margin-top: 12px`, no padding/border. Just text below the controls.

Mobile (`@media (max-width: 720px)`):
- Reduce gap to 10px.
- `.audio-time` becomes `flex-basis: 100%; text-align: right; margin-top: 8px;` so it wraps to its own line below the buttons + scrubber.

### JavaScript additions to `PAGE_JS`

Runs once at page load. Guards on `document.querySelector('.audio-el')` — no-op if no audio element on the page, so it's safe to ship on every briefing.

Pseudocode:

```js
const audio = document.querySelector('.audio-el');
if (audio) {
  const playBtn = document.querySelector('.audio-play');
  const playIcon = document.querySelector('.audio-icon-play');
  const backBtn = document.querySelector('.audio-back');
  const fwdBtn = document.querySelector('.audio-fwd');
  const track = document.querySelector('.audio-track');
  const progress = document.querySelector('.audio-progress');
  const buffered = document.querySelector('.audio-buffered');
  const curEl = document.querySelector('.audio-cur');
  const durEl = document.querySelector('.audio-dur');

  const fmt = (s) => {
    if (!isFinite(s)) return '--:--';
    const m = Math.floor(s / 60);
    const sec = Math.floor(s % 60).toString().padStart(2, '0');
    return `${m}:${sec}`;
  };

  playBtn.addEventListener('click', () => {
    if (audio.paused) { audio.play(); playIcon.textContent = '⏸'; playBtn.setAttribute('aria-label', 'Pause'); }
    else { audio.pause(); playIcon.textContent = '▶'; playBtn.setAttribute('aria-label', 'Play'); }
  });
  backBtn.addEventListener('click', () => { audio.currentTime = Math.max(0, audio.currentTime - 10); });
  fwdBtn.addEventListener('click', () => { audio.currentTime = Math.min(audio.duration || Infinity, audio.currentTime + 10); });

  audio.addEventListener('timeupdate', () => {
    const ratio = audio.duration ? (audio.currentTime / audio.duration) : 0;
    progress.style.width = (ratio * 100) + '%';
    curEl.textContent = fmt(audio.currentTime);
  });
  audio.addEventListener('loadedmetadata', () => { durEl.textContent = fmt(audio.duration); });
  audio.addEventListener('progress', () => {
    if (audio.buffered.length && audio.duration) {
      buffered.style.width = ((audio.buffered.end(audio.buffered.length - 1) / audio.duration) * 100) + '%';
    }
  });
  audio.addEventListener('ended', () => { playIcon.textContent = '▶'; playBtn.setAttribute('aria-label', 'Play'); });

  track.addEventListener('click', (e) => {
    const rect = track.getBoundingClientRect();
    const ratio = (e.clientX - rect.left) / rect.width;
    if (audio.duration) audio.currentTime = ratio * audio.duration;
  });

  document.addEventListener('keydown', (e) => {
    if (e.target.tagName === 'INPUT' || e.target.tagName === 'TEXTAREA') return;
    if (e.code === 'Space') { e.preventDefault(); playBtn.click(); }
    else if (e.code === 'ArrowLeft') { backBtn.click(); }
    else if (e.code === 'ArrowRight') { fwdBtn.click(); }
  });
}
```

### `build_briefings.py` changes

1. New constant near the existing `BRIEF_DIR`: `AUDIO_DIR = BRIEF_DIR / "audio"`.
2. `render_briefing()` gains a third parameter `audio_filename: str | None`. If non-`None`, the function builds the player section HTML (with `audio_filename` substituted into the `<audio src>`) and inserts it between the closing `</header>` of the brief-head and the opening `<main class="prose">`.
3. In `main()`, after computing `date_str`, check `(AUDIO_DIR / f"{date_str}-podcast.mp3").exists()`. If yes, pass `f"{date_str}-podcast.mp3"` to `render_briefing()`; otherwise pass `None`.
4. Append the new CSS rules listed above to `PAGE_CSS`.
5. Append the JS block listed above to `PAGE_JS`, after the existing theme-toggle code.

No new dependencies. No file beyond `build_briefings.py` itself is modified.

### `publish.sh` changes

After the existing markdown-copy block (around line 24), add:

```bash
SRC_MP3="$SRC_DIR/podcast/${DATE}-podcast.mp3"
if [[ -f "$SRC_MP3" ]]; then
  mkdir -p "$REPO_DIR/briefings/audio"
  /bin/cp -f "$SRC_MP3" "$REPO_DIR/briefings/audio/"
  echo "  copied $(basename "$SRC_MP3") → briefings/audio/"
fi
```

The existing `git add briefings/` already picks up the audio subdirectory recursively, so no other change to the script is needed.

## Hosting & assets

MP3 files live in `briefings/audio/` and are committed to the `profolio` GitHub repo. GitHub Pages serves them at `https://yillonmask.github.io/briefings/audio/<DATE>-podcast.mp3`.

Acceptable for now: ~7.6 MB per episode, well below GitHub's 100 MB per-file limit, ~1 GB soft repo limit reached after ~130 episodes if nothing is pruned. If repo size becomes a concern, episodes can be moved to GitHub Releases or external storage later without touching the player code — only the `<audio src>` interpolation in `build_briefings.py` would need to change.

## Error handling

- **MP3 missing for a date:** player block is not rendered. No JS runs (guarded on element presence).
- **`audio.duration` is `NaN` / `Infinity`:** time display shows `--:--`. Skip math falls back to `currentTime + 10` without clamping. Player remains functional.
- **Network error loading the MP3:** the browser surfaces this on the `<audio>` element. We don't render a custom error UI — the page is still readable as text.
- **Keyboard shortcuts inside form inputs:** guarded by checking `e.target.tagName`. The site has no forms today, but this prevents future regressions.

## Testing plan

1. **Local build:** run `python3 build_briefings.py`. Verify `2026-05-14-briefing.html` contains a `.brief-audio` section and the other dated HTML files do not.
2. **Direct browser open:** open `2026-05-14-briefing.html` via `file://`. Click play, pause, ±10s, click on the track to seek, press Space/←/→. Verify the time display updates and the progress bar fills as audio plays.
3. **Theme:** toggle dark mode. Verify card, buttons, progress bar, and caption all adapt to the dark palette without manual overrides.
4. **Mobile:** shrink browser to <720px. Verify time wraps to its own line and buttons remain laid out horizontally.
5. **End-to-end publish:** run `./publish.sh 2026-05-14`. Confirm MP3 lands in `briefings/audio/`, commit lands on remote. After ~30–60s, load `https://yillonmask.github.io/briefings/2026-05-14-briefing.html` and re-run step 2 against the live page.
6. **Negative case:** run `./publish.sh 2026-05-15` (no MP3 in podcast dir). Confirm no player block appears on the page, no JS errors in the console, no audio dir entry created for that date.
